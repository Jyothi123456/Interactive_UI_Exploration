Monoalphabetic Cipher:

  • A mono-alphabetic cipher (aka simple substitution cipher) is a substitution cipher where each letter of the plain text is replaced with another letter of the alphabet. It uses a fixed key which consist of the 26 letters of a “shuffled alphabet”.

    • Encryption Process:

    1. Key Generation: In the Monoalphabetic Cipher, the key is a mapping of each letter of the alphabet to another letter. This mapping defines the substitution pattern. For example, 'A' might be substituted with 'F', 'B' with 'X', and so on. The key needs to be kept secret between the sender and the receiver.
    2. Substitution: To encrypt a message, each letter of the plaintext is replaced with its corresponding letter according to the key. For example, if the key says 'A' is substituted with 'F', every 'A' in the plaintext will be replaced with 'F', and so on for the entire message.
    3. Result: The encrypted message, known as ciphertext, is the result of the substitution process. The ciphertext can then be transmitted over an insecure channel because it's unintelligible without knowledge of the key.

    Decryption Process:

    1. Key: The recipient of the ciphertext must possess the key, which defines the substitution pattern used during encryption.
    2. Reverse Substitution: Using the key, each letter in the ciphertext is replaced by its corresponding letter in the reverse direction. For example, if 'F' corresponds to 'A' in the key, every 'F' in the ciphertext is replaced with 'A', and so on.
    3. Result: The result of this reverse substitution process is the original plaintext message.

    Example:

    Let's say we have a key where 'A' is substituted with 'F', 'B' with 'X', and so on. If we want to encrypt the plaintext message "DIVYA" using this key:

    • D is substituted with 'I'
    • I is substituted with 'N'
    • V is substituted with 'A'
    • Y is substituted with 'D'
    • A is substituted with 'F'

    So, "DIVYA" becomes "INADF" in ciphertext. To decrypt "INADF" back to "DIVYA", we simply reverse the substitution using the same key:

    • I is substituted with 'D'
    • N is substituted with 'I'
    • A is substituted with 'V'
    • D is substituted with 'Y'
    • F is substituted with 'A'

    So, "INADF" decrypts back to "DIVYA".

    Strengths and Weaknesses:

    Strengths: It's easy to understand and implement. It's suitable for educational purposes and for basic encryption needs where security is not a primary concern.
    Weaknesses: The Monoalphabetic Cipher is vulnerable to frequency analysis, where the frequency of letters in the ciphertext can be used to deduce the original message. It's also susceptible to known-plaintext attacks, where an attacker can deduce the key if they have access to both the ciphertext and a corresponding portion of the plaintext.

    Caesar Cipher:

    What is Caesar Cipher: To encrypt a message using the Caesar Cipher, you shift each letter in the plaintext by a fixed number of positions down the alphabet. For example, if the fixed number (key) is 3, then each letter in the plaintext is replaced by the letter three positions down the alphabet.

    Encryption: To encrypt a message using the Caesar Cipher, you shift each letter in the plaintext by a fixed number of positions down the alphabet. For example, if the fixed number (key) is 3, then each letter in the plaintext is replaced by the letter three positions down the alphabet.

    Let's encrypt the message "JYOTHI" with a Caesar Cipher using a key of 3:

    Original Letter Encrypted Letter
    J O
    Y T
    O D
    T Y
    H M
    I N

    So, "JYOTHI" becomes "OTDYMN" after encryption with a key of 5.

    Decryption: Decryption is the reverse process of encryption. To decrypt a message encrypted with the Caesar Cipher, you shift each letter in the ciphertext back by the same number of positions that you used for encryption.

    Using the same key of 5, let's decrypt the ciphertext "OTDYMN":

    Encrypted Letter Decrypted Letter
    O J
    T Y
    D O
    Y T
    M H
    N I

    So, "OTDYMN" becomes "JYOTHI" after decryption with a key of 5.
